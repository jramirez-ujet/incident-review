AFFECTED AREAS
UJET · Voice / Agent Client SDK
WHAT HAPPENED

A client SDK regression caused inbound calls to fail 2 to 3 seconds after an agent answered. Each failed call requeued to the next agent and failed again, so callers eventually hung up. Outbound calling was unaffected. Assa Abloy reported the issue on July 9, and it recurred across seven tenants including Connective, Bump Health, Herschend, and Occam Global.

HOW IT WAS RESOLVED

Rerouting customer phone numbers off the affected voice path proved unreliable, so the team shipped client release v5.0.16, which reverts the SDK change behind the failures. The fix reached prd04 on July 14 and prd08 on July 15. Customers retested and confirmed inbound calling is working normally, with no new reports since.

FOLLOW-UP ITEMS
Release v5.0.16 to the remaining environments (prd03 / prd04 / prd19) (outstanding)

AFFECTED AREAS
UJET · Outbound / ALM Dialer Imports
WHAT HAPPENED

Keybank could not import ALM dialer lists into their UAT environment. Uploads of a roughly 44 MB file failed with out-of-memory errors in the import service, blocking the team from loading and processing new dialer lists.

HOW IT WAS RESOLVED

The memory limit on the import service was raised to 2 GB through change management and the service was redeployed. The failing file then imported successfully, and the team verified all records were present before closing.

FOLLOW-UP ITEMS
Rewrite CSV header processing to stream files instead of loading them into memory (outstanding)
Audit dialer import service deployments so memory limits and runtime versions match documented configuration (outstanding)
